## 分支策略

### 分支命名规范
```
feature/<功能名>-<阶段>
bugfix/<功能名>
hotfix/<issue>
docs/<功能名>
```

### 关键分支

| 分支 | 用途 | 生命周期 |
|---|---|---|
| `main` | 生产分支 | 永久 |
| `develop` | 开发分支 | 永久 |
| `feature/*` | 功能开发 | 临时 |
| `bugfix/*` | Bug 修复 | 临时 |
| `docs/*` | 文档更新 | 临时 |

## 具体分支规划

### 当前开发流程

```
# 1. 从 develop 创建功能分支
git checkout -b feature/pdf-parsing develop

# 2. 开发完成后合并到 develop
git checkout develop
git merge --no-ff feature/pdf-parsing
git push origin develop

# 3. 删除功能分支
git branch -d feature/pdf-parsing
```

### 推荐开发顺序 (考虑并行)

#### 第一批并行 (可同时开发)
- [ ] `feature/pdf-parsing` - 页面结构解析
- [ ] `feature/toc-extraction` - TOC 提取

#### 第二批 (依赖第一批)
- [ ] `feature/figure-detection` - 图表检测

#### 第三批 (可并行)
- [ ] `feature/llm-api` - LLM API
- [ ] `feature/chat-ui` - 聊天 UI
- [ ] `feature/streaming` - 流式响应

#### 第四批 (可并行)
- [ ] `feature/summary` - 摘要
- [ ] `feature/translation` - 翻译
- [ ] `feature/recommendation` - 推荐
- [ ] `feature/knowledge-base` - 知识库
